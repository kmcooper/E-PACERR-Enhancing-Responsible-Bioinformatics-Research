Bioinformatics Competency Self-Assessment

This self-check is designed to help you identify areas of confidence and areas where you may need more support before or during bioinformatics training.

How to use this self-assessment

Rate each statement using the scale below. Be honest: this is not graded. The goal is to help you figure out where you may be strong, where you are still developing, and where you may need additional instruction or practice.

  • 1 = I have no experience with this (Unaware)
  • 2 = I have heard of this, but I cannot do it yet (Aware)
  • 3 = I can do this with help (With help)
  • 4 = I can do this independently (Do)
  • 5 = I can explain or teach this to others (Teach)

1. Biological Foundations

1. I can explain how DNA, RNA, and proteins relate to common bioinformatics datasets.

2. I can connect a biological research question to the kinds of data needed to answer it.

2. Computing and Data Handling

3. I can work with common bioinformatics file types such as FASTA, FASTQ, CSV, or TSV.

4. I can use a command line or similar computing environment to run basic tools or scripts.

5. I can organize files, folders, and project materials in a way that others could follow.

3. Statistics and Interpretation

6. I can interpret common statistical ideas such as means, variation, p-values, and confidence intervals.

7. I can choose an appropriate basic statistical test for a simple biological question.

8. I can tell the difference between a result that is statistically significant and a result that is biologically meaningful.

4. Databases and Information Retrieval

9. I can use common bioinformatics databases such as NCBI, UniProt, GEO, or related resources to find information.

10. I understand the difference between keyword searching, information retrieval, and structured database querying.

11. I can retrieve and interpret metadata needed to understand where a dataset came from and how it was generated.

5. Reproducibility and Workflow

12. I can document enough of my analysis that another person could understand what I did.

13. I can explain why versioning, workflow tracking, and clear file naming matter for reproducibility.

14. I can evaluate whether a bioinformatics tool or pipeline is appropriate for a given task.

6. Communication and Collaboration

15. I can explain a bioinformatics result clearly to someone with less computational experience.

16. I can work across biology and computing perspectives without assuming that everyone shares the same background knowledge.

Your Results

Competency Area Average Score Interpretation

Suggested interpretation: averages of 4.0–5.0 indicate a relative strength, 3.0–3.9 indicate a developing area, and anything below 3.0 suggests a likely area where more support, practice, or instruction would help.

This self-assessment is intended as a reflective tool to help learners think about preparation, confidence, and support needs across common bioinformatics competency areas.
