A Focus on Terminology

In bioinformatics and biomedical informatics, terms like repeatability, reproducibility, and replicability are often used interchangeably in casual conversation. However, in computational disciplines, these terms can describe different relationships among:

  • the research team,
  • the experimental or computational setup,
  • the data, code, software, and other artifacts, and
  • the expected result.
Core idea: These terms are not just vocabulary words. They describe different levels of evidence that a scientific or computational result can be obtained again.

ACM's Definition

The Association for Computing Machinery (ACM) is a major professional organization in computing. Because bioinformatics and biomedical informatics rely heavily on computational methods, ACM terminology is especially useful when discussing computational artifacts, software, and reproducible research.

ACM publication guidance also emphasizes professional standards, research integrity, artifact review, and the public sharing of code and data where appropriate.

What is an artifact?

In computational research, an artifact is a digital object created by researchers for a study or generated by the study itself.

Artifacts may include:

  • software,
  • scripts used to run an analysis,
  • scripts used to generate figures or visualizations,
  • datasets,
  • intermediate files from a computational pipeline, and
  • workflow or pipeline configuration files.
Why artifacts matter: If the artifacts are missing, unclear, or unusable, it becomes much harder to repeat, reproduce, or replicate a computational result.

Repeatability

Repeatability refers to obtaining the same measurement or result with stated precision using the same team, same measurement procedure, same measuring system, same operating conditions, and same location across multiple trials.

In computational work, this means a researcher can reliably repeat their own computation. A researcher runs the same analysis script on the same dataset, using the same software environment and parameters, and obtains the same output each time.

How repeatability differs from related terms

Term Team Setup Computational meaning
Repeatability Same team Same experimental or computational setup The original researcher can reliably repeat their own computation.
Reproducibility Different team Same experimental or computational setup An independent group can obtain the same result using the original authors’ artifacts.
Replicability Different team Different experimental or computational setup An independent group can obtain the same result using artifacts they develop independently.

Note: Different disciplines sometimes use these terms differently. This course emphasizes the ACM-style distinctions because of their usefulness in computational research.

Why failure to repeat or reproduce can happen

Failure to obtain the same result does not always mean misconduct occurred. It may come from several sources, including:

  • disorganized or incomplete methodology,
  • missing code, data, parameters, or intermediate files,
  • software version differences,
  • lack of statistical analysis or incorrect statistical analysis,
  • computational environment differences, or
  • fraud or intentional misconduct in rare cases.
Important: The goal of repeatability and artifact sharing is not only to catch errors. It is also to strengthen trust, improve transparency, and support independent review.

Supporting integrity in the research ecosystem

Repeatability, reproducibility, and replicability are supported by research practices that make computational work easier to inspect and rerun.

These practices include:

  • creating clear standards and best practices,
  • using review processes that value reproducible work,
  • making data and code available when possible,
  • documenting software versions and computational environments,
  • describing parameters and decisions clearly, and
  • preserving artifacts so they can be audited or reused.

Reflection Questions

Reflection questions
  1. Why might repeatability be considered a first step before reproducibility or replicability?
  2. What artifacts would someone need to repeat one of your computational analyses?
  3. What is one reason a computation might fail to repeat even if no one did anything unethical?
  4. Which is stronger evidence for a finding: repeatability, reproducibility, or replicability? Why?
